# ปิด modal แก้วันที่ก่อนรอการบันทึก

## เป้าหมาย

เมื่อผู้ใช้กดยืนยันใน SweetAlert ระหว่างการแก้วันที่ ระบบต้องปิด modal เลือกวันที่ทันที เพื่อไม่ให้ผู้ใช้เห็น modal ค้างระหว่างรอการบันทึกผ่าน Google Drive/API ที่ใช้เวลานาน

## ขอบเขต

- แก้เฉพาะ flow การยืนยันวันที่ใน `TimelineDetail`
- ไม่เปลี่ยนกติกาตรวจวันหยุด วันก่อนหน้าหรือการยืนยันผลกระทบของตาราง
- ไม่เปลี่ยน API และไม่เปลี่ยนรูปแบบข้อมูลที่บันทึก

## แนวทางการทำงาน

1. เมื่อผู้ใช้กดปุ่มยืนยันใน dialog ผลกระทบ ระบบเก็บ `order` และวันที่ที่กำลังแก้ไว้ใน local context
2. ปิด modal เลือกวันที่และล้าง state เฉพาะของ modal ทันที
3. เรียก API ต่อด้วยค่าที่เก็บไว้ โดยส่ง `confirmShortening` และ `confirmOverwrite` ตามเดิม
4. แสดง success alert เมื่อบันทึกสำเร็จ
5. หากบันทึกล้มเหลว ให้แสดง error ระดับหน้าแทนการเปิด modal เดิมกลับมา เพราะผู้ใช้กดยืนยันการแก้ไขไปแล้ว
6. หากผู้ใช้กดยกเลิกใน dialog ผลกระทบ จะไม่ปิด modal เลือกวันที่และไม่เรียก API ซ้ำ

## การออกแบบโค้ด

แยกข้อมูลการแก้วันที่ออกจาก state ที่ควบคุมการ render modal เพื่อให้การบันทึกแบบ asynchronous ยังใช้ `order` และวันที่เดิมได้ แม้ `editingOrder` จะถูกล้างหลังปิด modal แล้ว หลีกเลี่ยงการเรียกฟังก์ชันซ้ำที่พึ่งพา state ซึ่งอาจถูก reset ไปแล้ว

## การทดสอบ

- เพิ่ม component test ที่จำลอง `onAdjustStep` ให้ค้างอยู่ระหว่างรอ Promise และตรวจว่า heading modal แก้วันที่หายไปทันทีหลังผู้ใช้กดยืนยันใน SweetAlert
- ตรวจว่าการเรียก API ยังได้รับ order, วันที่, version และ flags ครบถ้วน
- รัน test ที่เกี่ยวข้องและชุดตรวจสอบของโปรเจกต์เดิม
