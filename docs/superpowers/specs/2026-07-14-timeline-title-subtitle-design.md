# Timeline Title and Subtitle Design

## Goal

ปรับข้อความทุกแถวในหน้า Timeline ให้ตรงกับเอกสาร “ขั้นตอน Timeline” และแยกลำดับชั้นเป็นหัวข้อหลัก (`title`) กับคำอธิบายรอง (`subtitle`) โดยไม่เปลี่ยนข้อมูลที่จัดเก็บ

## Scope

- ใช้ข้อความจากแท็บ “ข้อความ ขั้นตอน” ของเอกสาร Google Docs
- แสดง title เด่นกว่า subtitle และใช้รูปแบบเดียวกันบนหน้าจอกับงานพิมพ์
- ขั้นตอนที่ไม่มี subtitle แสดงเฉพาะ title
- รองรับทั้งวงเงินไม่เกิน 5,000,000 บาทและวงเงินสูงกว่า 5,000,000 บาท
- คงวันที่ จำนวนวัน SLA การแก้วันที่ และข้อมูลโครงการเดิมทั้งหมด
- ไม่ commit และไม่ push Git

## Approach

เพิ่ม helper ฝั่งการแสดงผลใน `TimelineDetail` เพื่อแปลง label เดิมเป็น `{ title, subtitle }` ตามความหมายของขั้นตอน ไม่เปลี่ยน schema หรือข้อมูลในฐานข้อมูล วิธีนี้ทำให้ Timeline เดิมได้รับข้อความใหม่ทันทีและหลีกเลี่ยง data migration

## Verification

- Component tests ตรวจ title/subtitle ของทุกประเภทขั้นตอน
- ตรวจทั้งโครงการวงเงินเล็กและวงเงินสูงกว่า 5,000,000 บาท
- รัน focused tests, typecheck, lint และ build
