# Step 1 Heading Design

## Goal

ปรับการแสดงผลเฉพาะขั้นตอนที่ 1 เพื่อให้หัวข้อและรายละเอียดอ่านแยกกันชัดเจน โดยยังคงข้อมูลกำหนดการเดิมทั้งหมด

## Scope

- ใช้กับขั้นตอนที่มี `order` เท่ากับ 1 ในทุกประเภทวงเงิน
- แสดงหัวข้อหลัก `จัดทำเอกสาร` ด้วยน้ำหนักและขนาดที่เด่นกว่ารายละเอียด
- แสดงรายละเอียดในบรรทัดถัดไปด้วยข้อความ `รายงานขอซื้อขอจ้าง • แต่งตั้งคณะกรรมการ • ประกวดราคา`
- รายละเอียดใช้ขนาดตัวอักษรเล็กกว่าและสีรอง เพื่อรักษาลำดับชั้นของข้อมูล
- ใช้รูปแบบเดียวกันทั้งหน้าจอและงานพิมพ์
- ไม่แก้ label ที่จัดเก็บในโครงการ ไม่แก้วันที่ ระยะเวลา ลำดับขั้น หรือขั้นตอนอื่น

## Implementation Boundary

เพิ่มตัวแสดงผลเฉพาะขั้นตอนที่ 1 ใน `TimelineDetail` โดยแยกหัวข้อและรายละเอียดเป็นคนละ element ส่วนขั้นตอนอื่นใช้เส้นทางการแสดงผลเดิม เพื่อจำกัดผลกระทบและเปิดทางให้ผู้ใช้ประเมินตัวอย่างก่อนขยายรูปแบบไปยังขั้นตอนอื่น

## Error and Compatibility Behavior

- โครงการเดิมไม่ต้องย้ายข้อมูล เพราะเป็นการเปลี่ยนเฉพาะการแสดงผล
- ทุกประเภทวงเงินเห็นข้อความขั้นตอนที่ 1 เหมือนกันตามสเปกนี้
- การพิมพ์ยังซ่อนข้อความช่วยเหลือและปุ่มตามกฎเดิม แต่หัวข้อและรายละเอียดใหม่ต้องปรากฏครบ

## Verification

- Component test ยืนยันหัวข้อและรายละเอียดของขั้นตอนที่ 1
- Component test ยืนยันว่ารายละเอียดอยู่คนละบรรทัดและมีขนาดเล็กกว่าหัวข้อ
- Component test ยืนยันว่าขั้นตอนที่ 2 เป็นต้นไปยังใช้ข้อความเดิม
- รันชุดทดสอบเดิม, typecheck และ production build ก่อนเผยแพร่
